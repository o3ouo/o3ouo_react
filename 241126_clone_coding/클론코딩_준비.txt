완료된 파일 토대로 평가 

=진행순서=

1. 목표설정 : 어떤 웹사이트를 클론할 지 선택
참고 사이트 : gdweb, siteinspire
적절한 난이도 선택 : 자신의 기술 수준에 맞는 프로젝트를 선택

2. 자료수집 : 필요한 이미지, 아이콘, 폰트 등 준비
전체캡쳐 > 사이트 개발자툴 > ctrl + shift + p(명령어) > 원본 크기 스크린샷 캡쳐
폰트 다운 > 사이트 개발자툴 - 네트워크 - 폰트 - ctrl + shift + r

3. 분석 : 원본 사이트의 디자인과 사용자 경험 요소 분석

4. html/css 작성 : 작용 섹션을 단계적으로 구현
반응형 디자인 적용 (다양한 디바이스에 잘 작동하도록 미디어 쿼리를 추가

5. 기능 구현 : 자바스크립트 또는 라이브러리 사용해 동적 기능 추가

6. 테스트 및 디버깅 : 다양한 브라우저와 디바이스에서 테스트하여 호환성을 확인

7. 개선 및 확장 : 원본과 동일하게 구현한 후 자신만의 스타일이나 기능을 추가 (분석 때 뭔가 불편했던 부분을 개선)
아이디어로 기능을 추가하거나 개선을 필수

개선 시 제안서 작성 !
제안서 내용 :
- 배경설명 : 현재 웹페이지의 문제점이나 개선이 필요한 이유를 간략히 설명
- 목적 : 수정 작업을 통해 달성하고자 하는 목표를 명시
- 수정 : 어떤 식으로 개선 및 수정을 했는지 명시. 디자인 변경, 기능 추가, 기술적 업데이트
- 기대효과 : 수정작업 완료 후 예상되는 효과를 설명

8. 저작권 : footer > 상업적 목적이 아닌 교육용 공지

페이지 캡쳐 : 개발툴 - Ctrl + Shift + P

10. 발표
발표시 내용
- 목표설정 : ex) 웹 서비스의 기능을 학습하고 구현하는 것이 목표
- 기능선정 : 구현할 주요 기능 선정 그 이유
- 기술 : 사용한 기술 스택 / html / css / js / swiper / gasp /..
- 시연 : 완성된 프로젝트의 주요 기능을 시연 + 사용한 기술을 설명
- 개선 : 개선 했던 부분 > 제안서 내용
- kpt 회고 : keep(유지할 점), problem(문제점), try(시도할 점)
- 느낀점 : 100시간 동안 프로젝트를 진행하면서 느꼈던 것 + 좋았던 것 + 별로였던 것 + 성장 내용 등
