CSR에서는 초기 로드 시 정적 파일이 브라우저로 전달되고, 이후에는 브라우저에서동적으로 페이지를 렌더링한다. 이러한 특성 때문에 정적 파일을 제공하는 서버가 필요하며, AWS S3와 같은 정적 호스팅 서버가 자주 사용된다. 이러한 서비스는 서버 부하를 줄이고 성능을 최적화하는 데 도움을 준다. 결론 : 리액트를 호스팅해야 하는 이유는 사용자 접근성을 보장하고, 클라이언트 측 렌더링(CSR) 특성을 지원하며, URL 라우팅 및 성능 최적화를 가능하게 하기 위함이다. 이러한 과정을 통해 리액트 애플리케이션은 실제 서비스로 제공될 수 있다. : REACT 주요 특징 - 컴포넌트 기반 아키텍처 : UI를 독립적이고 재사용 가능한 컴포넌트로 분리하여 개발과 유지보수를 용이하게 한다. 각 컴포넌트는 자체 상태(state)와 생명주기(lifecycle)를 가질 수 있다. - Virtual DOM : 리액트는 실제 DOM 대신 가벼운 복사본인 Virtual DOM을 사용하여 변경된 부분만 효율적으로 업데이트한다. 이를 통해 성능을 최적화하고 빠른 렌더링을 지원한다. - JSX(javaScript XML) : HTML과 유사한 문법으로 UI를 정의할 수 있는 자바스크립트 확장 문법으로, 가독성과 생산성을 높인다. - 상태 관리와 생태계 : 리액트는 상태 관리와 라우팅 같은 기능을 위해 Redux, React Router 등과 쉽게 통합할 수 있는 방대한 생태계를 제공한다. : 컴포넌트(Component) ㄴ 화면을 구성하는 요소, UI를 구성하는 요소 ㄴ 재사용을 할 수 있다 : Virtual DOM - DOM을 자바스크립트 객체로 흉내낸 것으로 일종의 복제판 - React 업데이트가 발생하면 실제 DOM을 수정하기 전 이 가상 복제판 DOM에 먼저 반영해본다 ★★ Node.js ★★ 2008-2009년 쯤 개발 웹브라우저가 아닌 환경에서도 자바스크립트 코드를 실행시켜주는 자바스크립트 런타임, 즉 자바스크립트의 실행환(구동기) 구동기라는 건 기본적으로 무언가를 동작시키는 기계, 프로그램을 의미한다. 우리는 프로그래밍에서는 이러한 언어를 구동하는 구동기를 실행 환경이라고 해서 런타임이라고 표현한다. 공식 웹사이트에 들어가 보면 상단에 소개문으로 node.js는 자바스크립트 엔진으로 빌드, 만들어진 자바스크립트의 런타임이다. 라고 소개한다.
